<objective>
Deploy Rspamd spam filtering with greylisting backed by Redis, integrate with all three mail server options via milter protocol, and create integration test scripts for the complete home device mail stack.

Purpose: Spam filtering is essential before enabling catch-all addresses or exposing the mail server to the internet. Rspamd provides industry-standard spam scoring and greylisting reduces unsolicited messages by temporarily rejecting first-time senders (legitimate servers retry, spammers typically do not). The integration test scripts verify the entire mail pipeline works end-to-end and serve as the phase test suite per project memory rules.

Output: Rspamd + Redis containers, milter integration for all mail servers, greylisting configuration, and test scripts covering mail flow and spam filtering.
</objective>

**home-device/spam-filter/rspamd/local.d/greylist.conf:**
- servers = "redis:6379" (Redis connection for greylist state)
- timeout = 300 (5 minute greylist delay -- standard retry interval)
- expire = 86400 (24 hour expiry for greylist entries)
- greylist_min_score = 4.0 (only greylist messages with spam score >= 4.0 -- avoid greylisting clean mail)
- whitelisted_ip = "/etc/rspamd/local.d/whitelist_ip.map" (create this file with 10.0.0.0/8, 192.168.0.0/16, 172.16.0.0/12 for private networks including WireGuard)
- check_local = false (don't greylist mail from local networks -- important for cloud relay forwarded mail)
- check_authed = false (don't greylist authenticated users sending via submission)
- message = "Service temporarily unavailable, try again later" (4xx response message)
- report_time = true (tell sender when greylisting expires)

**home-device/spam-filter/rspamd/local.d/worker-proxy.conf:**
- milter = yes (enable milter protocol for mail server integration)
- bind_socket = "*:11332" (listen on all interfaces within Docker network)
- upstream "local" { default = yes; self_scan = yes; } (Rspamd scans locally)
- timeout = 120s (generous timeout for large messages)

**home-device/spam-filter/rspamd/local.d/actions.conf:**
- reject = 15 (reject messages with spam score >= 15)
- add_header = 6 (add X-Spam header for scores >= 6)
- greylist = 4 (greylist for scores >= 4, matches greylist_min_score)
- rewrite_subject = 12 (rewrite subject with [SPAM] prefix for scores >= 12)
- Comment: These thresholds are conservative defaults. Users can tune based on their spam volume.

Integrate Rspamd with all three mail server options and create the phase test suite.

**Stalwart milter integration (config.toml):**
- Add milter configuration section:
  ```toml
  [session.data.milter."rspamd"]
  enable = true
  hostname = "rspamd"
  port = 11332
  timeout = "10s"
  tempfail-on-error = true
  ```
- This applies to inbound SMTP (port 25) data phase. Stalwart's milter integration handles the connection to Rspamd automatically.
- Submission port (587) should NOT use milter -- authenticated users bypass spam filtering. Verify Stalwart's milter scoping: by default milter applies to all listeners. If Stalwart doesn't support per-listener milter scoping, add a condition to skip milter for authenticated sessions (check Stalwart docs for `if = "!is_authenticated"` or equivalent condition syntax).

**Maddy milter integration (maddy.conf):**
- Add milter check in the inbound SMTP (port 25) pipeline:
  ```
  smtp tcp://0.0.0.0:25 {
      destination $(local_domains) {
          check {
              rspamd tcp://rspamd:11332
          }
          deliver_to &local_mailboxes
      }
  }
  ```
- Do NOT add milter check to the submission (port 587) pipeline -- authenticated users bypass spam filtering.
- Maddy supports rspamd integration natively via the `check { rspamd ... }` directive (preferred over generic milter for Maddy).

**Postfix milter integration (main.cf):**
- Add to main.cf:
  ```
  # Rspamd milter integration (inbound SMTP only)
  smtpd_milters = inet:rspamd:11332
  non_smtpd_milters = inet:rspamd:11332
  milter_protocol = 6
  milter_default_action = accept
  milter_mail_macros = i {mail_addr} {client_addr} {client_name} {auth_authen}
  ```
- milter_default_action = accept means if Rspamd is down, mail still delivers (safe default -- better than losing mail)
- In master.cf, the submission entry (port 587) should override milters to empty to bypass Rspamd for authenticated users:
  ```
  submission inet n - n - - smtpd
    -o smtpd_milters=
    -o non_smtpd_milters=
  ```
  This ensures outbound mail from authenticated users is not spam-filtered.

**Phase test suite -- home-device/tests/test-mail-flow.sh:**
Create a comprehensive shell script that tests the full mail pipeline. This serves as the end-of-phase test suite per project memory rules.

The script should:
1. Detect which mail server profile is active (check running containers)
2. Test SMTP inbound delivery:
   - Use swaks (Swiss Army Knife for SMTP) or Python smtplib or raw netcat/telnet to send a test message to port 25
   - Verify message arrives in user's mailbox (check via IMAP using curl or openssl s_client)
3. Test IMAP access:
   - Connect to port 993 with TLS
   - Authenticate with test credentials
   - List mailboxes (INBOX should exist)
   - Check for the test message delivered in step 2
4. Test SMTP submission:
   - Connect to port 587 with STARTTLS
   - Authenticate with test credentials
   - Send a test message (should be accepted for relay)
5. Test multi-user isolation:
   - Send messages to two different users
   - Verify each user only sees their own messages via IMAP
6. Test alias delivery:
   - Send to alias address
   - Verify message appears in target user's IMAP inbox
7. Test catch-all:
   - Send to random undefined address at catch-all domain
   - Verify message appears in catch-all user's IMAP inbox

The script should:
- Accept command-line args for host, port, user, password (with sane defaults for Docker localhost)
- Use swaks if available, fall back to Python smtplib, then to openssl s_client
- Use curl with IMAP URL for IMAP checks (curl supports imaps://)
- Output clear PASS/FAIL for each test
- Exit with non-zero code if any test fails
- Include usage instructions in comments

**home-device/tests/test-spam-filter.sh:**
Test Rspamd integration:
1. Verify Rspamd is running: curl http://localhost:11334/stat (Rspamd web UI stats endpoint)
2. Verify Redis is reachable from Rspamd: check Rspamd logs for Redis connection success
3. Send a clean message via SMTP to port 25 and verify it's delivered (not rejected or greylisted with score < 4)
4. Send a message with GTUBE pattern (Generic Test for Unsolicited Bulk Email -- a standard test string that Rspamd recognizes as spam):
   - GTUBE string: XJS*C4JDBQADN1.NSBN3*2IDNEN*GTUBE-STANDARD-ANTI-UBE-TEST-EMAIL*C.34X
   - Verify message is rejected or marked as spam (X-Spam header added)
5. Verify greylisting state in Redis: check that greylist entries exist after a greylisted message attempt
6. Verify authenticated submission bypasses Rspamd: send via port 587 with auth, check no spam headers added
