  <action>
Build the ephemeral storage verification system and optimize the container image.

1. **cloud-relay/relay/ephemeral/verify.go** - Ephemeral storage verification:
   - `VerifyResult` struct: `Clean` (bool), `Violations` ([]Violation), `ScannedPaths` ([]string), `ScannedAt` (time.Time)
   - `Violation` struct: `Path` (string), `Size` (int64), `ModTime` (time.Time), `Type` (string: "queue_file", "data_file", "temp_file")
   - `VerifyNoPersistedMail(postfixQueueDir string) (*VerifyResult, error)` — scans Postfix queue directories for lingering mail files:
     a. Check `/var/spool/postfix/incoming/` — should be empty after forwarding
     b. Check `/var/spool/postfix/active/` — should be empty after forwarding
     c. Check `/var/spool/postfix/deferred/` — should be empty (deferred means forwarding failed)
     d. Check `/var/spool/postfix/hold/` — should be empty (held messages are persisted)
     e. Check `/var/spool/postfix/corrupt/` — should be empty
     f. Scan Go daemon's working directory for any temp files
   - Ignore Postfix control files (pid, master.lock, etc.) — only flag actual message data files
   - `WatchAndVerify(ctx context.Context, interval time.Duration, queueDir string, alertFunc func(VerifyResult))` — periodic verification loop that calls alertFunc when violations found. Run every 60 seconds by default.
   - Return file count, total size, and oldest file age for any violations found

2. **cloud-relay/relay/ephemeral/verify_test.go** - Tests:
   - Test VerifyNoPersistedMail with empty queue dirs (clean result)
   - Test VerifyNoPersistedMail with files in incoming/ (violation detected)
   - Test VerifyNoPersistedMail with files in deferred/ (violation detected)
   - Test that Postfix control files (pid, master.lock) are NOT flagged as violations
   - Use os.MkdirTemp to create fake queue directory structures for testing

3. **Optimize cloud-relay/Dockerfile** for minimum image size:
   - Builder stage: use `golang:1.24-alpine` (NOT golang:1.24 which is Debian-based and huge)
   - Strip binary: `CGO_ENABLED=0 GOOS=linux go build -ldflags="-s -w"` (already specified in 02-01)
   - Consider UPX compression if binary exceeds 5MB: `apk add upx && upx --best relay-daemon`
   - Runtime stage: `FROM alpine:3.21` with `--no-cache` on all apk add
   - Minimize layers: combine RUN commands where possible
   - Remove apk cache: `rm -rf /var/cache/apk/*` (though --no-cache handles this)
   - Pin package versions for reproducibility: `postfix=~3.9` or whatever Alpine 3.21 ships
   - Add `.dockerignore` file in repo root to exclude `.git/`, `.planning/`, `docs/`, test files from build context
   - Target breakdown: Alpine base ~5MB + Postfix+deps ~15MB + wireguard-tools ~5MB + Go binary ~3-5MB + gettext ~2MB = ~30-32MB
   - Add labels: `LABEL org.opencontainers.image.source=https://github.com/trek-e/darkpipe`

4. **Update cloud-relay/docker-compose.yml** — Add resource limits:
   - `deploy.resources.limits.memory: 256M` (enforce 256MB RAM limit per requirement)
   - `deploy.resources.reservations.memory: 128M`
   - Add `RELAY_EPHEMERAL_CHECK_INTERVAL=60` env var for periodic verification
