    Create the monitoring/cert/ package for certificate lifecycle automation:

    **monitoring/cert/watcher.go:**
    - `type CertInfo struct` with Path, Subject, NotBefore, NotAfter time.Time, DaysLeft int, Lifetime time.Duration, ShouldRenew bool, ShouldWarn bool, ShouldCritical bool
    - `type CertWatcher struct` with certPaths []string, warnDays int (14), criticalDays int (7), renewalFraction float64 (2.0/3.0)
    - `NewCertWatcher(paths []string) *CertWatcher` - defaults: warnDays=14, criticalDays=7, renewalFraction=2/3
    - `CheckCert(certPath string) (*CertInfo, error)` - use crypto/x509 to parse PEM, extract NotAfter, calculate daysLeft, determine shouldRenew based on 2/3 lifetime rule
    - `CheckAll() ([]CertInfo, error)` - check all registered certificate paths
    - `ShouldRenew(cert *CertInfo) bool` - true if elapsed time > 2/3 of lifetime. Per user decision: for 90-day cert, renew at 60 days (30 remaining). For 45-day cert, renew at 30 days (15 remaining). For step-ca internal: configurable.
    - Per user decision: Let's Encrypt timeline awareness (90-day now, 45-day opt-in May 2026, 64-day Feb 2027, 45-day Feb 2028). Implementation: the 2/3 fraction handles all these automatically.

    **monitoring/cert/rotator.go:**
    - `type RenewalType string` with `LetsEncrypt`, `StepCA`, `SelfSigned` constants
    - `type RotatorConfig struct` with CertName, RenewalType, DryRun bool, StagingMode bool (for LE testing per Pitfall 3)
    - `RenewWithRetry(config RotatorConfig) error` - uses cenkalti/backoff/v4 with MaxRetries=3
      - For LetsEncrypt: exec `certbot renew --cert-name <name>` (add --dry-run if DryRun, --staging if StagingMode)
      - For StepCA: exec `step ca renew <cert> <key> --force`
      - For SelfSigned: exec openssl to regenerate self-signed cert
    - `isPermanentError(output []byte) bool` - check for ACME account issues, invalid domain errors (don't retry these)
    - Per user decision: retry with exponential backoff (3 retries), alert admin on failure, keep using old cert until actual expiry
    - `RenewIfNeeded(watcher *CertWatcher, certPath string, config RotatorConfig) (renewed bool, err error)` - combines check + renew

    **monitoring/cert/dkim.go:**
    - `type DKIMRotationConfig struct` with Domain, Prefix (default "darkpipe"), KeyDir, DNSProviderType (cloudflare/route53/manual), LastRotationPath string
    - `RotateDKIM(config DKIMRotationConfig) error` - orchestrates quarterly DKIM rotation:
      1. Check if rotation is due using dns/dkim.ShouldRotate()
      2. Generate new selector using dns/dkim.GenerateSelector() for next quarter
      3. Generate new 2048-bit RSA key using dns/dkim.GenerateKey() (reuse Phase 4)
      4. Write new key to KeyDir with new selector filename
      5. Write DNS record instructions to stdout (or call DNS provider API if configured)
      6. Update signing configuration to use new selector
      7. Log: keep old key for 7-day overlap period (per Pitfall 6: dual-key overlap)
      8. Update lastRotation timestamp file
    - `ShouldRotateDKIM(lastRotationPath string) (bool, error)` - read last rotation timestamp, compare with dns/dkim.ShouldRotate()
    - Per user decision: automated quarterly rotation matching Phase 4 selector format {prefix}-{YYYY}q{Q}

    **monitoring/cert/reload.go:**
    - `type ServiceReloader struct` with services []ServiceConfig
    - `type ServiceConfig struct` with Name, ReloadCommand string (e.g., "postfix reload", "caddy reload")
    - `ReloadServices(services []ServiceConfig) error` - execute reload commands sequentially
    - Per Claude's discretion: hot reload via SIGHUP/reload command (not restart) to avoid service interruption
    - Default services: Postfix (postfix reload), Caddy (caddy reload --config /etc/caddy/Caddyfile)
    - `GenerateSystemdPathUnit(certPaths []string, reloadService string) string` - generate systemd .path unit content for cert file monitoring (PathChanged= directives)
