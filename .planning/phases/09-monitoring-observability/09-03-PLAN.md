  <action>
    Create the monitoring/status/ package that aggregates all monitoring data and provides user interfaces:

    **monitoring/status/aggregator.go:**
    - `type SystemStatus struct` with Health HealthSummary, Queue QueueSummary, Delivery DeliverySummary, Certificates CertSummary, Timestamp time.Time, OverallStatus string ("healthy"/"degraded"/"unhealthy")
    - `type HealthSummary struct` with Status string, Checks []CheckDetail (from monitoring/health)
    - `type QueueSummary struct` with Depth, Deferred, Stuck int (from monitoring/queue)
    - `type DeliverySummary struct` with Delivered, Deferred, Bounced, Total int, RecentEntries []DeliveryEntry (from monitoring/delivery)
    - `type CertSummary struct` with Certificates []CertDetail, NextExpiry time.Time, DaysUntilExpiry int (from monitoring/cert)
    - `type StatusAggregator struct` with health *health.Checker, queue func() (*queue.QueueStats, error), delivery *delivery.DeliveryTracker, certs *cert.CertWatcher
    - `NewStatusAggregator(health, queue, delivery, certs)` - dependency injection
    - `GetStatus(ctx context.Context) (*SystemStatus, error)` - collect from all sources, compute overall status
    - Overall status logic: all healthy = "healthy", any warn = "degraded", any critical = "unhealthy"
    - JSON serializable (all structs have json tags) for --json CLI output

    **monitoring/status/cli.go:**
    - `func RunStatusCommand(args []string)` - parse flags and display status
    - Flags: `--json` (JSON output for scripting/Home Assistant), `--watch` (auto-refresh every 5 seconds, per Claude's discretion: one-shot default with --watch option), `--watch-interval` (customize refresh interval)
    - Human-readable output format (default):
      ```
      DarkPipe System Status
      ======================
      Overall: HEALTHY

      Services:
        Postfix SMTP:  OK
        IMAP Server:   OK
        WireGuard:     OK

      Mail Queue:
        Depth:    3
        Deferred: 0
        Stuck:    0

      Recent Deliveries (last 24h):
        Delivered: 47
        Deferred:  2
        Bounced:   0

      Certificates:
        relay.example.com:  62 days remaining (OK)
        internal CA cert:   89 days remaining (OK)
      ```
    - JSON output (--json): dump full SystemStatus struct
    - Watch mode (--watch): clear terminal and redisplay every N seconds (use ANSI clear)
    - CLI warning: check /data/monitoring/cli-alerts.json for pending alerts, display at bottom: "WARNING: 2 pending alerts. Run 'darkpipe alerts' to view."
    - Use fatih/color (already in project) for colored output: green=OK, yellow=WARN, red=ERROR

    **monitoring/status/push.go:**
    - `type HealthchecksPinger struct` with checkURL string, client *http.Client, interval time.Duration
    - `NewHealthchecksPinger(checkURL string, interval time.Duration) *HealthchecksPinger` - default interval 5 minutes
    - `Ping(ctx context.Context) error` - GET request to check URL with 10s timeout
    - `PingWithStatus(ctx context.Context, status *SystemStatus) error` - POST request with status JSON body (Healthchecks.io supports this)
    - `Run(ctx context.Context)` - background goroutine, ping every interval, no error on individual failures (Healthchecks.io alerts on missed pings = Dead Man's Switch pattern)
    - Configure with MONITOR_HEALTHCHECK_URL env var (optional, disabled if empty)
    - Per user decision: push-based pings to external uptime services via outbound HTTP (no inbound exposure)

    **monitoring/status/aggregator_test.go:**
    - Test GetStatus with all-healthy dependencies returns "healthy"
    - Test GetStatus with failing health check returns "unhealthy"
    - Test GetStatus with cert warning returns "degraded"
    - Use mock implementations of health, queue, delivery, cert interfaces

    **monitoring/status/cli_test.go:**
    - Test RunStatusCommand with --json flag outputs valid JSON
    - Test human-readable format contains expected sections (Services, Queue, Deliveries, Certificates)
    - Test --watch flag is recognized (can't easily test terminal refresh)

    **monitoring/status/push_test.go:**
    - Test Ping sends GET request to configured URL (use httptest.NewServer)
    - Test PingWithStatus sends POST with JSON body
    - Test Run cancels cleanly on context cancellation
    - Test disabled when URL is empty (no-op)
