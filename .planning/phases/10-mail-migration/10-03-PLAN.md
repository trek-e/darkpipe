**provider.go** — Provider interface and registry:
- `Provider` interface:
  - `Name() string` — Display name ("Gmail", "Outlook", etc.)
  - `Slug() string` — CLI identifier ("gmail", "outlook", etc.)
  - `ConnectIMAP(ctx context.Context) (*imapclient.Client, error)` — Authenticate and return IMAP client
  - `ConnectCalDAV(ctx context.Context) (*caldav.Client, error)` — Return CalDAV client or nil if unsupported
  - `ConnectCardDAV(ctx context.Context) (*carddav.Client, error)` — Return CardDAV client or nil if unsupported
  - `SupportsLabels() bool` — Gmail-style labels vs folders
  - `SupportsAPI() bool` — Has provider-specific API (MailCow/Mailu)
  - `SupportsCalDAV() bool`
  - `SupportsCardDAV() bool`
  - `GetFolderMapping() map[string]string` — Provider-specific folder mappings (delegates to mailmigrate.FolderMapper defaults)
  - `GetSkipFolders() map[string]bool` — Folders to skip
  - `IMAPEndpoint() string` — IMAP server:port
  - `CalDAVEndpoint() string`
  - `CardDAVEndpoint() string`
  - `WizardPrompts() []WizardPrompt` — Provider-specific prompts for wizard (e.g., OAuth flow, API key, app password guidance)
- `WizardPrompt` struct: Type (string: "oauth", "input", "info"), Label (string), Field (string), HelpText (string)
- `Registry` — map[string]Provider with `Register(p Provider)` and `Get(slug string) (Provider, error)` and `List() []string`
- Global `DefaultRegistry` initialized with all providers

**oauth.go** — Shared OAuth2 device authorization grant flow:
- `OAuthConfig` struct: ProviderName (string), ClientID (string), Scopes ([]string), Endpoint (oauth2.Endpoint), DeviceAuthURL (string)
- `RunDeviceFlow(ctx context.Context, config *OAuthConfig, display func(verificationURL, userCode string)) (*oauth2.Token, error)`:
  1. Create oauth2.Config from OAuthConfig
  2. Call config.DeviceAuth(ctx) to get device code response
  3. Call display callback with verification URL and user code (for wizard to show to user)
  4. Call config.DeviceAccessToken(ctx, response) — this polls automatically per RFC 8628
  5. Return token (includes AccessToken and possibly RefreshToken)
- `TokenSourceFromToken(config *OAuthConfig, token *oauth2.Token) oauth2.TokenSource` — Create auto-refreshing token source
- `XOAUTH2Token(email, accessToken string) string` — Format XOAUTH2 SASL string for IMAP authentication (`user=<email>\x01auth=Bearer <token>\x01\x01`)
- Note: Gmail and Outlook both require XOAUTH2 SASL mechanism for IMAP

**gmail.go** — Gmail provider:
- `GmailProvider` struct: Email (string), Token (*oauth2.Token), ClientID (string), ClientSecret (string)
- Implement all Provider interface methods
- `ConnectIMAP`: DialTLS to imap.gmail.com:993, authenticate with XOAUTH2 SASL using access token
- `ConnectCalDAV`: Return caldav.Client pointing to https://www.googleapis.com/caldav/v2/ with OAuth2 bearer auth via http.Client with token source
- `ConnectCardDAV`: Return carddav.Client pointing to https://www.googleapis.com/.well-known/carddav with OAuth2 bearer auth
- `SupportsLabels() = true`
- `GetFolderMapping()`: Gmail-specific mappings per locked decision ([Gmail]/Sent Mail -> Sent, etc.)
- `GetSkipFolders()`: [Gmail]/All Mail, [Gmail]/Important, [Gmail]/Starred
- `WizardPrompts()`: Return OAuth device flow prompt
- Gmail OAuth2 scopes: `https://mail.google.com/` (full IMAP), `https://www.googleapis.com/auth/calendar` (CalDAV), `https://www.googleapis.com/auth/contacts` (CardDAV)
- Note: ClientID/ClientSecret should come from user's GCP project or environment variables (GMAIL_CLIENT_ID, GMAIL_CLIENT_SECRET). Document this requirement.

**outlook.go** — Outlook provider:
- `OutlookProvider` struct: Email (string), Token (*oauth2.Token), ClientID (string)
- `ConnectIMAP`: DialTLS to outlook.office365.com:993, authenticate with XOAUTH2 SASL
- `ConnectCalDAV`: Return nil (Outlook CalDAV not widely available; document that users should export .ics from Outlook web)
- `ConnectCardDAV`: Return nil (same limitation; suggest .vcf export)
- Outlook OAuth2: Device auth URL `https://login.microsoftonline.com/common/oauth2/v2.0/devicecode`, scopes `https://outlook.office.com/IMAP.AccessAsUser.All offline_access`
- `GetFolderMapping()`: Outlook-specific (Deleted Items -> Trash, Sent Items -> Sent, Junk Email -> Junk, Clutter -> skip)
- `WizardPrompts()`: Return OAuth device flow prompt
- Note: Microsoft requires OAuth2 for IMAP after April 2026 (per research). Basic auth will not work.

**icloud.go** — iCloud provider:
- `iCloudProvider` struct: Email (string), AppPassword (string)
- `ConnectIMAP`: DialTLS to imap.mail.me.com:993, login with email + app password
- `ConnectCalDAV`: Return caldav.Client pointing to https://caldav.icloud.com/ with basic auth
- `ConnectCardDAV`: Return carddav.Client pointing to https://contacts.icloud.com/ with basic auth
- `SupportsLabels() = false`
- `GetFolderMapping()`: minimal (iCloud uses standard folder names)
- `WizardPrompts()`: Return "info" type prompt explaining how to create app-specific password at appleid.apple.com, then "input" prompts for email and app password

**mailcow.go** — MailCow API provider (success criteria #3):

- MailCow API client functions (all use net/http with X-API-Key header):
  - `GetMailboxes(ctx context.Context) ([]MailCowMailbox, error)` — GET {APIURL}/api/v1/get/mailbox/all
  - `GetAliases(ctx context.Context) ([]MailCowAlias, error)` — GET {APIURL}/api/v1/get/alias/all
  - `GetDomains(ctx context.Context) ([]MailCowDomain, error)` — GET {APIURL}/api/v1/get/domain/all
- `MailCowMailbox` struct: Username (string), Name (string), QuotaUsed (int64), Messages (int), Active (bool)
- `MailCowAlias` struct: Address (string), GoTo (string), Active (bool)
- `MailCowDomain` struct: DomainName (string), Active (bool)
- `ExportConfig(ctx context.Context) (*MailCowExport, error)` — Aggregate all API data for migration preview
- `MailCowExport` struct: Mailboxes ([]MailCowMailbox), Aliases ([]MailCowAlias), Domains ([]MailCowDomain)
- `WizardPrompts()`: "input" prompts for API URL and API key, then standard IMAP credentials
- `GetFolderMapping()`: Standard (MailCow uses standard IMAP folder names)

**mailu.go** — Mailu provider:
- `MailuProvider` struct: APIURL (string), APIKey (string), IMAPHost (string)
- `SupportsAPI() = true` — Mailu has REST API for user/alias/domain export
- `ConnectIMAP`: DialTLS to IMAPHost:993, login with credentials
- Mailu API client functions:
  - `GetUsers(ctx context.Context) ([]MailuUser, error)` — GET {APIURL}/api/v1/user (uses Authorization: Bearer header)
  - `GetAliases(ctx context.Context) ([]MailuAlias, error)` — GET {APIURL}/api/v1/alias
  - `GetDomains(ctx context.Context) ([]MailuDomain, error)` — GET {APIURL}/api/v1/domain
- Fallback: If API fails (older Mailu versions), fall back to standard IMAP-only mode with a warning
- `WizardPrompts()`: "input" prompts for API URL and API key (optional), IMAP credentials
- `GetFolderMapping()`: Standard

**dockermailserver.go** — docker-mailserver provider:
- `DockerMailServerProvider` struct: IMAPHost (string), Username (string), Password (string)
- `SupportsAPI() = false` — No API, standard IMAP only
- `ConnectIMAP`: DialTLS to IMAPHost:993, login
- `ConnectCalDAV`: Return nil (docker-mailserver has no CalDAV)
- `ConnectCardDAV`: Return nil
- `WizardPrompts()`: "input" prompts for IMAP host, username, password
- `GetFolderMapping()`: Standard

**generic.go** — Generic IMAP/CalDAV/CardDAV provider:
- `GenericProvider` struct: IMAPHost (string), IMAPPort (int), Username (string), Password (string), CalDAVURL (string), CardDAVURL (string), UseTLS (bool)
- Supports all protocols based on what the user provides
- `ConnectIMAP`: DialTLS or DialStartTLS based on config
- `ConnectCalDAV`: If CalDAVURL provided, create client with basic auth; else nil

- `WizardPrompts()`: "input" prompts for IMAP host/port/user/pass, optional CalDAV URL, optional CardDAV URL
- `GetFolderMapping()`: Empty (pass-through, user can override with --folder-map)
