**state.go** — Migration state tracking with JSON persistence:
- `MigrationState` struct with fields: Version (string), Provider (string), StartedAt (time.Time), LastUpdated (time.Time), MailMessages (map[string]bool keyed by Message-ID hash), CalEvents (map[string]bool keyed by event UID), Contacts (map[string]bool keyed by contact email hash), Errors ([]MigrationError for skip-and-report tracking)
- `MigrationError` struct: Folder (string), Subject (string), Date (time.Time), Error (string)
- `HashMessageID(messageID string) string` — SHA-256 hash of Message-ID for state map keys
- `FallbackHash(from, subject string, date time.Time) string` — SHA-256 of From+Subject+Date for messages without Message-ID (per pitfall #5)
- `LoadState(path string) (*MigrationState, error)` — Load from JSON file, return empty state if file doesn't exist
- `SaveState(path string) error` — Persist to JSON file with 0600 permissions, update LastUpdated
- `IsMessageMigrated(messageID string) bool` — Check by hash
- `MarkMessageMigrated(messageID string)` — Mark by hash
- `IsCalEventMigrated(uid string) bool` and `MarkCalEventMigrated(uid string)`
- `IsContactMigrated(email string) bool` and `MarkContactMigrated(email string)`
- `AddError(folder, subject string, date time.Time, err error)` — Record skipped message
- `Summary() (migrated int, skipped int, errors []MigrationError)` — Return counts for end-of-migration summary
- State file path default: `/data/migration-state.json` (per locked decision)

**mapping.go** — Configurable folder mapping:
- `FolderMapper` struct with fields: Mappings (map[string]string), SkipFolders (map[string]bool)
- `NewFolderMapper(provider string, overrides map[string]string) *FolderMapper` — Initialize with provider-specific defaults, apply user overrides
- Provider defaults per locked decisions:
  - Gmail: `[Gmail]/Sent Mail` -> `Sent`, `[Gmail]/Drafts` -> `Drafts`, `[Gmail]/Trash` -> `Trash`, `[Gmail]/Spam` -> `Junk`; skip `[Gmail]/All Mail`, `[Gmail]/Important`, `[Gmail]/Starred`
  - Outlook: `Deleted Items` -> `Trash`, `Clutter` -> skip, `Sent Items` -> `Sent`, `Junk Email` -> `Junk`
  - Generic/others: No default mapping (pass-through)
- `Map(sourceFolder string) (destFolder string, skip bool)` — Returns mapped name and whether to skip
- `AllMappings() map[string]string` — For dry-run preview display
- `LabelToKeyword(label string) string` — Sanitize Gmail label to valid IMAP keyword atom (replace spaces with underscores, remove special chars)
- `LabelsAsFolders` bool field on FolderMapper — When true, map each label to a subfolder instead of keyword (--labels-as-folders fallback for Dovecot/Maddy)

**imap.go** — IMAP sync engine:
- `IMAPSync` struct with fields: Source (*imapclient.Client), Dest (*imapclient.Client), State (*MigrationState), Mapper (*FolderMapper), StatePath (string), LabelsAsKeywords (bool), OnProgress (func(folder string, current, total int)), OnFolderStart (func(folder string, total int)), OnFolderDone (func(folder string)), BatchSize (int, default 50)
- `NewIMAPSync(source, dest *imapclient.Client, state *MigrationState, mapper *FolderMapper, statePath string) *IMAPSync`
- `ListSourceFolders(ctx context.Context) ([]FolderInfo, error)` — List all source mailboxes with message counts. `FolderInfo` struct: Name (string), Messages (uint32), mapped destination name, skip flag. Uses IMAP LIST command.
- `SyncFolder(ctx context.Context, sourceFolder string) (*FolderResult, error)` — Sync a single folder:
  1. Map source folder name to destination via FolderMapper; skip if mapped to ""
  2. Select source folder (read-only)
  3. Fetch all message UIDs
  4. For each UID (in batches of BatchSize):
     a. Fetch message: envelope (for Message-ID, From, Subject, Date), flags, full body (RFC822)
     b. Extract Message-ID from envelope; if empty, use FallbackHash
     c. Check state.IsMessageMigrated(); skip if already done
     d. Create destination folder if it doesn't exist (IMAP CREATE)
     e. APPEND to destination with original flags and INTERNALDATE (per locked decision for date preservation)
     f. Mark message migrated in state; call OnProgress callback
     g. On error: add to state.Errors, log, continue (skip-and-report per locked decision)
  5. Save state periodically (every BatchSize messages) and at end
  6. Return FolderResult: folder name, migrated count, skipped count, error count
- `FolderResult` struct: Folder (string), Migrated (int), Skipped (int), Errors (int)
- `SyncAll(ctx context.Context) (*SyncResult, error)` — Sync all non-skipped folders sequentially. Returns SyncResult with per-folder results and totals.
- `SyncResult` struct: Folders ([]FolderResult), TotalMigrated (int), TotalSkipped (int), TotalErrors (int)
- `DryRun(ctx context.Context) (*DryRunResult, error)` — Preview mode: list folders with message counts and proposed mappings. No messages transferred. DryRunResult: Folders ([]DryRunFolder with Name, Messages, MappedTo, WillSkip fields), TotalMessages (int)
- Handle IMAP APPEND options: use `imap.AppendOptions{Flags: flags, Time: internalDate}` per research Pattern 4
- For Gmail label support: if LabelsAsKeywords is true and source supports X-GM-LABELS, attempt to fetch labels and apply as IMAP keywords on destination (best-effort, log warning if extension not supported by go-imap v2)
- Rate limiting: add configurable delay between batches (default 0, useful for Gmail quota limits)
