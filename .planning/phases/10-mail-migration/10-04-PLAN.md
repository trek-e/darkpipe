**migrate_cmd.go** — Cobra migrate command:
- `migrateCmd` as cobra.Command:
  - Use: "migrate"
  - Short: "Migrate email, contacts, and calendars from an existing provider"
  - Long: Describes migration capabilities, lists supported providers
- Flags:
  - `--from` (string, required): Provider slug (gmail, outlook, icloud, mailcow, mailu, docker-mailserver, generic)
  - `--apply` (bool, default false): Execute migration (without this, dry-run only per locked decision)
  - `--folder-map` (string): Custom folder mappings in "Source:Dest,Source2:Dest2" format
  - `--labels-as-folders` (bool, default false): Map Gmail labels to folders instead of keywords (Dovecot/Maddy fallback)
  - `--state-file` (string, default "/data/migration-state.json"): Path to state file
  - `--contacts-mode` (string, default "append"): Contact merge mode (append/overwrite/skip)
  - `--imap-dest` (string, default from DarkPipe config): Destination IMAP server
  - `--caldav-dest` (string, default from DarkPipe config): Destination CalDAV server
  - `--carddav-dest` (string, default from DarkPipe config): Destination CardDAV server
  - `--vcf-file` (string): Path to .vcf file for contact import
  - `--ics-file` (string): Path to .ics file for calendar import
  - `--batch-size` (int, default 50): IMAP fetch batch size
  - `--dest-user` (string): Destination mailbox username
  - `--dest-pass` (string): Destination mailbox password
- RunE function:
  1. If --from not provided, list supported providers and exit
  2. Get provider from registry
  3. Call wizard.RunMigrationWizard with provider, flags, and config
- Register migrateCmd in main.go: `rootCmd.AddCommand(versionCmd, setupCmd, migrateCmd)` — import migrate_cmd.go's init function or add directly

**migrate_wizard.go** — Interactive migration wizard:
- `MigrationConfig` struct: Provider (providers.Provider), Apply (bool), FolderMap (map[string]string), LabelsAsFolders (bool), StatePath (string), ContactsMode (string), DestIMAP (string), DestCalDAV (string), DestCardDAV (string), DestUser (string), DestPass (string), VCFFile (string), ICSFile (string), BatchSize (int)
- `RunMigrationWizard(ctx context.Context, cfg *MigrationConfig) error`:
  1. Display header: "Mail Migration - {Provider.Name()}"
  2. Run provider-specific wizard prompts (from Provider.WizardPrompts()):
     - For "oauth" type: call RunOAuthDeviceFlow (see oauth_flow.go)
     - For "input" type: prompt user with survey.Input
     - For "info" type: display information text (e.g., iCloud app password instructions)
  3. Connect to source: call Provider.ConnectIMAP/CalDAV/CardDAV
  4. If destination credentials not provided via flags, prompt for them
  5. Connect to destination IMAP/CalDAV/CardDAV
  6. Load migration state from StatePath
  7. Create IMAPSync, CalDAVSync, CardDAVSync instances with state and folder mapper
  8. **DRY-RUN PHASE** (always runs first):
     - Call IMAPSync.DryRun → show table of folders with message counts and proposed mappings
     - If Provider.SupportsCalDAV(): call CalDAVSync.DryRun → show calendar counts
     - If Provider.SupportsCardDAV(): call CardDAVSync.DryRun → show contact counts
     - If VCFFile provided: call DryRunVCF → show contact file preview
     - If ICSFile provided: call DryRunICS → show calendar file preview
     - If MailCow/Mailu API: show exported users/aliases/domains table
     - Display grand total: "{N} messages in {M} folders, {C} calendar events, {D} contacts"
  9. If not Apply: display "Run with --apply to execute migration" and exit
  10. If Apply: prompt for confirmation ("Migrate {N} messages, {C} events, {D} contacts?")
  11. **MIGRATION PHASE:**
     - Create MigrationProgress instance for progress display
     - Run IMAPSync.SyncAll with progress callbacks wired to MigrationProgress
     - Run CalDAVSync.SyncAll (if applicable)
     - Run CardDAVSync.SyncAll (if applicable)
     - Run ImportVCF/ImportICS (if files provided)
  12. **SUMMARY PHASE:**
     - Display migration summary table: migrated/skipped/errors per data type
     - If errors: display error details (folder, subject, date, error message)
     - Display state file location for resume information
     - If MailCow API was used: display user/alias creation instructions for DarkPipe

**progress.go** — pterm-based progress display:
- `MigrationProgress` struct wrapping pterm.MultiPrinter
- `NewMigrationProgress() *MigrationProgress`
- `Start()` — Initialize multi-printer
- `StartFolder(folder string, totalMessages int)` — Create named progress bar for folder
- `UpdateFolder(folder string, count int)` — Increment folder progress
- `CompleteFolder(folder string)` — Mark folder bar done, increment overall
- `SetOverall(totalFolders int)` — Set overall progress total
- `Stop()` — Clean up multi-printer
- Graceful degradation: if terminal doesn't support multi-printer (detect with os.Getenv("TERM") or isatty), fall back to simple log-style output: "[3/12] Migrating INBOX: 142/1203 messages"

**oauth_flow.go** — OAuth2 device flow UI:
- `RunOAuthDeviceFlow(ctx context.Context, provider providers.Provider) (*oauth2.Token, error)`:
  1. Call provider's OAuth config
  2. Call oauth.RunDeviceFlow with display callback that uses pterm:
     - Display verification URL in a highlighted box
     - Display user code in large, bold text
     - Show spinner while waiting for authorization
  3. On success: display "Authorization successful!" with checkmark
  4. Return token for provider to use
