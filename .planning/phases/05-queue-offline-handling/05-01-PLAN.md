Create the `cloud-relay/relay/queue/` package with three files:

**encrypt.go** — Age encryption wrapper:
- `func GenerateIdentity() (*age.X25519Identity, error)` — generates new age keypair
- `func LoadOrCreateIdentity(path string) (*age.X25519Identity, error)` — loads identity from file or creates new one (0600 permissions). On first startup generates and persists to mounted volume. Per research pitfall #5, losing this key means losing queued messages.
- `func Encrypt(data []byte, recipient age.Recipient) ([]byte, uint32, error)` — encrypts data with age, returns ciphertext + CRC32 checksum. Use `age.Encrypt()` streaming API writing to bytes.Buffer, then `crc32.ChecksumIEEE()` on result.
- `func Decrypt(encrypted []byte, checksum uint32, identity age.Identity) ([]byte, error)` — verifies CRC32 first (fast rejection of corrupted data), then decrypts with `age.Decrypt()`. If CRC32 passes but age.Decrypt fails (Poly1305 MAC failure), return specific error for quarantine logging (pitfall #7).
- Import `filippo.io/age` and `hash/crc32` (stdlib).

**queue.go** — In-memory encrypted message queue:
- `type QueuedMessage struct` with fields: ID (string, Message-ID), From (string), To ([]string), EnqueuedAt (time.Time), EncryptedData ([]byte), Checksum (uint32), InOverflow (bool), OverflowKey (string), Size (int64).
- `type MessageQueue struct` with fields: mu (sync.RWMutex), messages (map[string]*QueuedMessage keyed by Message-ID), order ([]string for FIFO ordering), recipient (age.Recipient), identity (age.Identity), maxRAMBytes (int64, default 200MB — leaves 56MB for OS+Postfix+Go within 256MB container limit), currentRAMBytes (int64), snapshotPath (string), maxMessages (int, default 10000 — cost control per research open question #2), ttlHours (int, default 168 — 7 days per research recommendation).
- `func NewMessageQueue(cfg QueueConfig) (*MessageQueue, error)` — initializes queue, loads or creates age identity from cfg.KeyPath.
- `func (q *MessageQueue) Enqueue(ctx context.Context, from string, to []string, data io.Reader) error` — reads message data, extracts Message-ID using `net/mail` stdlib (simpler than go-message for just header extraction; use `mail.ReadMessage()` then `msg.Header.Get("Message-Id")`). If Message-ID empty, generate fallback via SHA-256 hash (first 16 bytes, formatted as `<fallback-{hex}@darkpipe.local>`). Check dedup map. Encrypt message with age. If currentRAMBytes + msgSize > maxRAMBytes, return `ErrQueueFull` (let overflow handler in Plan 05-02 hook in, or reject). Add to messages map and order slice. Update currentRAMBytes.
- `func (q *MessageQueue) Dequeue(id string) (*QueuedMessage, []byte, error)` — removes message, decrypts, returns plaintext. Updates currentRAMBytes.
- `func (q *MessageQueue) Len() int` — returns queue length.
- `func (q *MessageQueue) RAMUsage() int64` — returns currentRAMBytes.
- `func (q *MessageQueue) PurgeExpired() int` — removes messages older than TTL. Returns count purged.
- `func (q *MessageQueue) Snapshot() error` — atomic JSON write of queue metadata to snapshotPath (temp file + fsync + rename per research pitfall #8). Only writes metadata (ID, From, To, EnqueuedAt, Size, InOverflow, OverflowKey) — encrypted data stays in memory.
- `func (q *MessageQueue) Close() error` — writes final snapshot.
- `var ErrQueueFull = errors.New("queue: RAM limit exceeded")` — exported error for overflow detection.
- `type QueueConfig struct` with fields: KeyPath (string), MaxRAMBytes (int64), MaxMessages (int), TTLHours (int), SnapshotPath (string).
- Note: Use `net/mail` stdlib for Message-ID extraction (avoids adding emersion/go-message dependency — research suggested it but net/mail is sufficient for just header parsing). Generate fallback ID with `crypto/sha256`.

**processor.go** — Background queue drain:
- `func (q *MessageQueue) StartProcessor(ctx context.Context, transport forward.Forwarder, interval time.Duration)` — goroutine that ticks every `interval` (default 30 seconds). On each tick: if queue empty, skip. Otherwise call processQueue().
- `func (q *MessageQueue) processQueue(ctx context.Context, transport forward.Forwarder)` — iterate queue in FIFO order (use order slice). For each message: decrypt, attempt delivery via `transport.Forward()` with exponential backoff (cenkalti/backoff/v4, MaxElapsedTime=2min, InitialInterval=1s, MaxInterval=15s). On success: remove from queue. On first transport failure: stop processing remaining messages (home device likely still offline — avoid hammering). Rate limit: process at most 10 messages per tick to prevent thundering herd (pitfall #3).
- `func (q *MessageQueue) processSingle(ctx context.Context, transport forward.Forwarder, msg *QueuedMessage) error` — decrypt + forward single message. Used by processQueue.
- Also run PurgeExpired() every tick to clean old messages.

**forward/queued.go** — QueuedForwarder that implements the Forwarder interface:
- `type QueuedForwarder struct` with fields: transport (Forwarder — underlying WireGuard or mTLS), queue (*queue.MessageQueue), enabled (bool — QUEUE-03 toggle).
- `func NewQueuedForwarder(transport Forwarder, q *queue.MessageQueue, enabled bool) *QueuedForwarder`
- `func (f *QueuedForwarder) Forward(ctx context.Context, from string, to []string, data io.Reader) error`:
  1. Read data into bytes.Buffer (io.Reader not seekable, need copy for potential queue).
  2. Attempt `f.transport.Forward(ctx, from, to, bytes.NewReader(buf.Bytes()))`.
  3. If success: return nil (fast path, no queuing).
  4. If error and `!f.enabled`: return the error directly (Postfix handles 4xx retry per QUEUE-03 — when relay daemon returns error, Postfix sends 451 to sender's MTA).
  5. If error and `f.enabled`: call `f.queue.Enqueue(ctx, from, to, bytes.NewReader(buf.Bytes()))`. If enqueue succeeds return nil (message accepted, will deliver later). If enqueue fails (ErrQueueFull), return the original transport error (triggers 4xx from Postfix).
- `func (f *QueuedForwarder) Close() error` — close transport, then queue.
- `func (f *QueuedForwarder) Queue() *queue.MessageQueue` — accessor for processor startup in main.go.

```go
// Initialize queue if enabled
var activeForwarder forward.Forwarder
if cfg.QueueEnabled {
    log.Println("Initializing encrypted message queue...")
    queueCfg := queue.QueueConfig{
        KeyPath:      cfg.QueueKeyPath,
        MaxRAMBytes:  cfg.QueueMaxRAMBytes,
        MaxMessages:  cfg.QueueMaxMessages,
        TTLHours:     cfg.QueueTTLHours,
        SnapshotPath: cfg.QueueSnapshotPath,
    }
    msgQueue, err := queue.NewMessageQueue(queueCfg)
    if err != nil {
        log.Fatalf("Failed to initialize message queue: %v", err)
    }

    queuedFwd := forward.NewQueuedForwarder(forwarder, msgQueue, true)
    activeForwarder = queuedFwd

    // Start background queue processor
    processorCtx, processorCancel := context.WithCancel(context.Background())
    defer processorCancel()
    go msgQueue.StartProcessor(processorCtx, forwarder, 30*time.Second)

    log.Printf("Queue enabled: max_ram=%dMB max_messages=%d ttl=%dh",
        cfg.QueueMaxRAMBytes/(1024*1024), cfg.QueueMaxMessages, cfg.QueueTTLHours)
} else {
    log.Println("Queue disabled: mail will bounce when home device is offline")
    activeForwarder = forwarder
}

// Use activeForwarder instead of forwarder for SMTP server
server := smtp.NewServer(activeForwarder, cfg)
```
