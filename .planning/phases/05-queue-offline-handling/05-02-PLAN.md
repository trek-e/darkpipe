**overflow.go** — S3-compatible overflow storage using minio-go:
- `type OverflowStorage struct` with fields: client (*minio.Client), bucketName (string).
- `func NewOverflowStorage(endpoint, accessKey, secretKey, bucket string, useSSL bool) (*OverflowStorage, error)` — creates MinIO client with `minio.New(endpoint, &minio.Options{Creds: credentials.NewStaticV4(accessKey, secretKey, ""), Secure: useSSL})`. Verify bucket exists with `client.BucketExists(ctx, bucket)`. If bucket doesn't exist, create it with `client.MakeBucket(ctx, bucket, minio.MakeBucketOptions{})`.
- `func (s *OverflowStorage) Upload(ctx context.Context, key string, data []byte) error` — uploads already-encrypted data (age encryption happens before upload in queue.go, not here). Use `client.PutObject()` with ContentType "application/octet-stream". Key format: `darkpipe/queue/{message-id-hash}` (sanitize Message-ID to filesystem-safe characters).
- `func (s *OverflowStorage) Download(ctx context.Context, key string) ([]byte, error)` — downloads encrypted data. Use `client.GetObject()` + `io.ReadAll()`.
- `func (s *OverflowStorage) Delete(ctx context.Context, key string) error` — removes object after successful delivery. Use `client.RemoveObject()`.
- `func (s *OverflowStorage) List(ctx context.Context, prefix string) ([]string, error)` — lists objects with prefix for recovery. Use `client.ListObjects()`.
- Import `github.com/minio/minio-go/v7` and `github.com/minio/minio-go/v7/pkg/credentials`.

**overflow_test.go** — Tests with mock/interface approach:

**queue.go modifications** — Add overflow support:
- Add `overflow *OverflowStorage` field to MessageQueue struct (nil if overflow disabled).
- Add `SetOverflow(overflow *OverflowStorage)` method to inject overflow after construction.
- Modify `Enqueue()`: When currentRAMBytes + msgSize > maxRAMBytes:
  1. If overflow is nil: return ErrQueueFull (existing behavior).
  2. If overflow is set: call `overflowOldestMessages(ctx, needed)` to spill oldest N messages to S3 until enough RAM is freed, then enqueue new message in RAM.
- Add `func (q *MessageQueue) overflowOldestMessages(ctx context.Context, needed int64) error` — iterate order slice from oldest. For each message not already in overflow: upload encrypted data to S3 via overflow.Upload(), set InOverflow=true and OverflowKey, nil out EncryptedData (free RAM), reduce currentRAMBytes. Stop when enough space freed.
- Modify processor `processQueue()`: For messages with InOverflow=true, call overflow.Download() to retrieve encrypted data before decryption. After successful delivery, call overflow.Delete() to clean up S3.
