  <action>
    Create a new Go module at home-device/profiles/ with module path `github.com/darkpipe/darkpipe/profiles`.

    **go.mod dependencies:**
    - github.com/groob/plist (for Plan 08-01 Task 2)
    - github.com/skip2/go-qrcode (for Plan 08-02)
    - github.com/google/uuid (UUIDv7 for profile IDs)
    - golang.org/x/crypto/bcrypt (password hashing)

    **generator.go — App password generation:**
    - `GenerateAppPassword() (string, error)` — Generate 16-character password using crypto/rand, charset `ABCDEFGHJKLMNPQRSTUVWXYZ23456789` (no confusing 0/O/1/I), formatted as `XXXX-XXXX-XXXX-XXXX` with hyphens for readability.
    - `HashPassword(password string) (string, error)` — Bcrypt hash with cost 12 (using golang.org/x/crypto/bcrypt).
    - `VerifyPassword(password, hash string) bool` — Bcrypt comparison.
    - Always use crypto/rand, never math/rand.

    **store.go — Storage interface:**
    ```go
    type AppPassword struct {
        ID          string    // UUIDv7
        Email       string    // User email (e.g., user@example.com)
        DeviceName  string    // Human label (e.g., "iPhone 15", "Thunderbird Desktop")
        HashedPassword string // Bcrypt hash
        CreatedAt   time.Time
        LastUsedAt  time.Time // Updated on successful auth
    }

    type Store interface {
        Create(email, deviceName, plainPassword string) (*AppPassword, error)
        List(email string) ([]AppPassword, error)
        Revoke(id string) error
        Verify(email, plainPassword string) (bool, error)
    }
    ```

    **stalwart.go — Stalwart backend:**
    - Implement Store interface using Stalwart's REST API for app password management.
    - Stalwart stores app passwords in `$app$<device-name>$<bcrypt-hash>` format.
    - HTTP requests to Stalwart admin API (configurable base URL, default `http://stalwart:8080`).
    - Authentication via admin credentials (from environment: `STALWART_ADMIN_USER`, `STALWART_ADMIN_PASSWORD`).

    **dovecot.go — Postfix+Dovecot backend:**
    - Implement Store interface using a JSON file store at a configurable path (default `/data/app-passwords.json`).
    - Password verification: Dovecot passdb configured to check this file via a script or passdb-extra lookup.
    - File locked with flock for concurrent access safety.
    - Passwords stored as bcrypt hashes.

    **maddy.go — Maddy backend:**
    - Implement Store interface using a JSON file store (same pattern as Dovecot backend).
    - Maddy configured with custom auth backend that checks app passwords first.
    - Same file-based approach as Dovecot for simplicity.

    **generator_test.go — Tests:**
    - Test password generation produces correct format (4 groups of 4 chars separated by hyphens).
    - Test password charset (only contains expected characters).
    - Test 100 passwords are all unique (no duplicates from crypto/rand).
    - Test HashPassword and VerifyPassword roundtrip.
    - Test VerifyPassword fails for wrong password.
    - Test GenerateAppPassword returns error handling (mock crypto/rand failure if possible).

    **payloads.go — Apple .mobileconfig payload structs:**
    - Define typed structs for all payload types using plist struct tags:
      - `EmailPayload` (com.apple.mail.managed) — IMAP on 993 (SSL), SMTP on 587 (STARTTLS)
      - `CalDAVPayload` (com.apple.caldav.account) — CalDAV with principal URL
      - `CardDAVPayload` (com.apple.carddav.account) — CardDAV with principal URL
      - `MobileConfigProfile` — Top-level profile container with PayloadContent array
    - Use typed structs (not map[string]interface{}) for type safety and testability.
    - Per user decision: profiles are UNSIGNED for v1 (research recommends this over self-signed).

    **generator.go — Profile generation:**
    ```go
    type ProfileConfig struct {
        Domain       string // e.g., example.com
        MailHostname string // e.g., mail.example.com
        Email        string // e.g., user@example.com
        AppPassword  string // Generated app password
        CalDAVURL    string // Empty if no CalDAV (optional)
        CardDAVURL   string // Empty if no CardDAV (optional)
        CalDAVPort   int    // 5232 for Radicale, 443 for Stalwart
        CardDAVPort  int    // 5232 for Radicale, 443 for Stalwart
    }

    type ProfileGenerator struct{}

    func (g *ProfileGenerator) GenerateProfile(cfg ProfileConfig) ([]byte, error)
    ```
    - Uses groob/plist for serialization (NOT map[string]interface{}, use typed structs from payloads.go).
    - Uses google/uuid for PayloadUUID (UUIDv7 if available, UUIDv4 fallback).
    - PayloadIdentifier format: `com.darkpipe.<domain>.<payload-type>`.
    - PayloadOrganization: "DarkPipe".
    - Per user decision: includes Email + CalDAV + CardDAV in ONE profile.
    - CalDAV/CardDAV payloads only included when CalDAVURL/CardDAVURL are non-empty.
    - Content-Type for serving: `application/x-apple-aspen-config`.

    **autoconfig.go — Mozilla Thunderbird autoconfig XML:**
    - `GenerateAutoconfig(domain, mailHostname string) ([]byte, error)` using text/template.
    - XML follows Mozilla autoconfig spec v1.1: https://wiki.mozilla.org/Thunderbird:Autoconfiguration
    - IMAP incoming server: port 993, SSL, password-cleartext auth, username=%EMAILADDRESS%.
    - SMTP outgoing server: port 587, STARTTLS, password-cleartext auth, username=%EMAILADDRESS%.
    - Served at: `/.well-known/autoconfig/mail/config-v1.1.xml` and `autoconfig.<domain>/mail/config-v1.1.xml`.
    - Per discretion: public endpoints (no auth required) for maximum client compatibility.

    **autodiscover.go — Microsoft Outlook autodiscover XML:**
    - `GenerateAutodiscover(email, mailHostname string) ([]byte, error)` using text/template.
    - Response XML follows Microsoft Autodiscover POX protocol.
    - IMAP setting: port 993, SSL, SPA=off.
    - SMTP setting: port 587, TLS (STARTTLS), SPA=off.
    - Served at: `autodiscover.<domain>/autodiscover/autodiscover.xml`.
    - Per discretion: public endpoints for maximum compatibility.

    - autodiscover: Generate XML, parse back, verify IMAP/SMTP settings correct.
