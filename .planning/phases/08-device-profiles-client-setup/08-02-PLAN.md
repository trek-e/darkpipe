    **token.go — Single-use token management:**
    - `GenerateSecureToken() (string, error)` — 32 bytes from crypto/rand, base64url-encoded (no padding). 256-bit entropy per NIST recommendations.
    - `TokenStore` interface:
      ```go
      type TokenStore interface {
          Create(email string, expiresAt time.Time) (token string, err error)
          Validate(token string) (email string, valid bool, err error)
          Invalidate(token string) error
          Cleanup() // Remove expired tokens
      }
      ```
    - `MemoryTokenStore` — In-memory implementation with sync.RWMutex for thread safety.
    - Token struct: `{Email, Token, ExpiresAt, Used bool, CreatedAt}`.
    - Validate checks: token exists, not expired, not already used. On successful validation, marks token as used IMMEDIATELY (before returning) to prevent race conditions.
    - Default expiry: 15 minutes.
    - Cleanup goroutine runs every 5 minutes to purge expired tokens.
    - Per user decision: QR codes are single-use -- once scanned and redeemed, token is invalidated.

    **generator.go — QR code generation:**
    - `GenerateQRCode(profileBaseURL, email string, store TokenStore) (qrURL string, err error)` — Creates single-use token, returns the URL to embed in QR code. URL format: `https://<profileBaseURL>/profile/download?token=<token>`.
    - `GenerateQRCodePNG(url string, size int) ([]byte, error)` — Generate QR code as PNG bytes using skip2/go-qrcode with Medium error correction. Default size 256x256.
    - `GenerateQRCodeTerminal(url string) (string, error)` — Generate QR code as ASCII art for CLI display (using go-qrcode's ToString method).
    - Per discretion: QR codes contain URL (not inline settings) per research recommendation. URL-based is single-use, auditable, and revocable.

    **handlers.go — Profile server HTTP handlers:**
    ```go
    type ProfileHandler struct {
        ProfileGen    *mobileconfig.ProfileGenerator
        AutoconfigGen func(domain, hostname string) ([]byte, error)
        AutodiscoverGen func(email, hostname string) ([]byte, error)
        TokenStore    qrcode.TokenStore
        AppPassStore  apppassword.Store
        Config        ServerConfig
    }
    ```
    - `GET /profile/download?token=<token>` — Validate token (single-use), look up email from token, generate new app password for device "QR-<timestamp>", generate .mobileconfig with app password embedded, serve with Content-Type `application/x-apple-aspen-config` and Content-Disposition `attachment; filename="darkpipe-mail.mobileconfig"`. If token invalid/expired/used, return 401 with helpful error message.
    - `GET /mail/config-v1.1.xml?emailaddress=<email>` — Return Thunderbird autoconfig XML for the domain. The `emailaddress` parameter is optional (Thunderbird sends it). No authentication required (public endpoint per discretion).
    - `GET /autodiscover/autodiscover.xml` — Return Outlook autodiscover XML. Accept POST with XML body containing email address (Outlook protocol). No authentication required.
    - `GET /health` — Returns 200 OK for Docker healthcheck.
    - `GET /qr/generate?email=<email>` — Authenticated endpoint (Basic Auth with admin credentials) that generates a QR code and returns PNG image. Used by webmail iframe.
    - `GET /qr/image?email=<email>` — Same as above but returns the QR code image inline (for embedding in webmail page).

    **main.go — Profile server entry point:**
    - HTTP server listening on configurable port (default 8090).
    - Load config from environment: `PROFILE_SERVER_PORT`, `MAIL_DOMAIN`, `MAIL_HOSTNAME`, `CALDAV_URL`, `CARDDAV_URL`, `MAIL_SERVER_TYPE` (stalwart/maddy/postfix-dovecot), `STALWART_ADMIN_USER`, `STALWART_ADMIN_PASSWORD`, `ADMIN_USER`, `ADMIN_PASSWORD`.
    - Initialize appropriate apppassword.Store based on MAIL_SERVER_TYPE.
    - Initialize ProfileGenerator, AutoconfigGen, AutodiscoverGen from 08-01 packages.
    - Graceful shutdown on SIGTERM/SIGINT.
    - Log requests to stdout (JSON format for container logging).

    **srv.go — RFC 6186 SRV record generation:**
    - `type SRVRecord struct { Service, Proto, Domain string; Priority, Weight, Port uint16; Target string }`
    - `GenerateSRVRecords(domain, mailHostname string) []SRVRecord` — Generate standard email SRV records per RFC 6186:
      - `_imaps._tcp.<domain>` → priority 0, weight 1, port 993, target mailHostname (IMAPS, preferred)
      - `_imap._tcp.<domain>` → priority 10, weight 0, port 143, target "." (unavailable, DarkPipe only supports TLS)
      - `_submission._tcp.<domain>` → priority 0, weight 1, port 587, target mailHostname
    - `GenerateAutodiscoverCNAME(domain, relayHostname string) []DNSRecord` — Generate CNAME records:
      - `autoconfig.<domain>` CNAME → `<relayHostname>`
      - `autodiscover.<domain>` CNAME → `<relayHostname>`
    - `func (r SRVRecord) String() string` — Format as DNS zone file line: `_imaps._tcp.example.com. IN SRV 0 1 993 mail.example.com.`
